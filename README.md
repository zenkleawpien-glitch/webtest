# FIREGAME SHOP — เว็บไซต์ + ระบบหลังบ้าน

ระบบขายไอดีเกม Free Fire พร้อมหน้าร้านและหลังบ้านสำหรับจัดการสินค้า

## สิ่งที่มีในระบบ

- **หน้าร้าน** (`/`) — แสดงรายการไอดี กดดูรายละเอียด มีปุ่มติดต่อซื้อทาง Facebook
- **หลังบ้าน** (`/admin.html`) — ล็อกอินด้วยรหัสผ่าน แล้วเพิ่ม/แก้ไข/ลบสินค้า เปลี่ยนสถานะ (พร้อมขาย / ขายแล้ว / ซ่อน) ใส่รูปภาพแบบลิงก์ URL
- ข้อมูลสินค้าเก็บในไฟล์ `data/products.json` (ไม่ต้องติดตั้งฐานข้อมูลแยก)

---

## วิธีติดตั้งและรัน (ครั้งแรก)

### 1. ติดตั้ง Node.js
ถ้ายังไม่มี ให้โหลดจาก https://nodejs.org (เลือกเวอร์ชัน LTS) แล้วติดตั้งตามปกติ

ตรวจสอบว่าติดตั้งสำเร็จด้วยคำสั่ง:
```
node -v
npm -v
```

### 2. ติดตั้ง dependencies
เปิด terminal/command prompt ไปที่โฟลเดอร์โปรเจกต์นี้ แล้วรัน:
```
npm install
```

### 3. เปิดเซิร์ฟเวอร์
```
npm start
```
หรือ
```
node server.js
```

จะเห็นข้อความแบบนี้:
```
========================================
 สร้างรหัสผ่านแอดมินเริ่มต้นแล้ว
 รหัสผ่าน: admin1234
 (กรุณาเปลี่ยนรหัสผ่านในหน้า /admin หลังล็อกอินครั้งแรก)
========================================
FIREGAME SHOP server is running:
  หน้าร้าน:   http://localhost:3000/
  หลังบ้าน:   http://localhost:3000/admin.html
```

**รหัสผ่านเริ่มต้นคือ `admin1234`** — ข้อความนี้จะแสดงแค่ครั้งแรกที่รันเท่านั้น (ตอนที่ยังไม่มีไฟล์ `data/config.json`)

### 4. เปิดเว็บ
- หน้าร้าน: เปิดเบราว์เซอร์ไปที่ `http://localhost:3000/`
- หลังบ้าน: ไปที่ `http://localhost:3000/admin.html` แล้วล็อกอินด้วยรหัสผ่านด้านบน

**สำคัญ: หลังล็อกอินครั้งแรก ให้กดปุ่ม "เปลี่ยนรหัสผ่าน" ที่หลังบ้านทันที เพื่อความปลอดภัย**

## วิธีแชร์ให้เพื่อนใช้งานจริง

ถ้าต้องการให้เพื่อนเปิดเว็บได้จากลิงก์สาธารณะ แนะนำให้ deploy ขึ้นโฮสต์ Node.js เช่น Render, Railway หรือ Fly.io

### แนะนำวิธีที่ง่ายสุด: Render

1. สร้าง repository นี้บน GitHub ให้เรียบร้อยแล้ว
2. เข้า Render แล้วสร้าง `New Web Service`
3. เลือกเชื่อมกับ repo นี้
4. ตั้งค่า
  - Build Command: `npm install`
  - Start Command: `npm start`
5. ตั้ง Environment Variables ถ้าต้องการ
  - `SESSION_SECRET` = ค่าสุ่มยาว ๆ
  - `ADMIN_PASSWORD` = รหัสผ่านแอดมินเริ่มต้นที่ต้องการ
6. Deploy แล้วเอา URL ที่ได้ส่งให้เพื่อนได้เลย

### ข้อควรรู้ก่อนแชร์

- ตอนนี้ข้อมูลสินค้าเก็บในไฟล์ `data/products.json` ถ้าโฮสต์ฟรีรีเซ็ตเครื่อง ข้อมูลอาจหายเมื่อ redeploy
- ถ้าจะใช้งานจริงระยะยาว ควรย้ายข้อมูลไปฐานข้อมูลถาวร เช่น PostgreSQL หรือ SQLite ที่ผูกกับดิสก์ถาวร
- ไฟล์ `data/config.json` ถูก ignore ไว้แล้ว จึงไม่ควรอัปขึ้น GitHub

## วิธีใช้งานหลังบ้าน

- **เพิ่มไอดีใหม่**: กดปุ่ม "เพิ่มไอดีใหม่" มุมขวาบน กรอกข้อมูล แล้วกดบันทึก
- **แก้ไขสินค้า**: กดไอคอนดินสอในตารางสินค้า
- **ลบสินค้า**: กดไอคอนถังขยะ จะมีหน้าต่างยืนยันก่อนลบจริง
- **เปลี่ยนสถานะอย่างรวดเร็ว**: ใช้ dropdown ในคอลัมน์ "สถานะ" ของตารางได้เลยโดยไม่ต้องเปิดฟอร์มแก้ไข
  - **พร้อมขาย** — แสดงในหน้าร้าน กดดูรายละเอียดและซื้อได้ปกติ
  - **ขายแล้ว** — ยังแสดงในหน้าร้าน แต่ขึ้นป้าย "ขายแล้ว" และกดซื้อไม่ได้
  - **ซ่อน** — ไม่แสดงในหน้าร้านเลย (เก็บไว้ในระบบเฉยๆ)
- **รูปภาพสินค้า**: ใส่เป็นลิงก์ URL รูปภาพ (เช่น อัปโหลดรูปขึ้น Facebook โพสต์/อัลบั้ม หรือเว็บฝากรูปอย่าง Imgur แล้วคัดลอกลิงก์รูปมาวาง) ถ้าไม่ใส่รูป ระบบจะแสดงไอคอนเปลวไฟแทนอัตโนมัติ

## การแก้ไขลิงก์ Facebook สำหรับติดต่อซื้อ

เปิดไฟล์ `public/index.html` ค้นหาบรรทัด:
```js
const FB_LINK = "https://facebook.com/yourfireGameShop";
```
เปลี่ยนเป็นลิงก์เพจ/โปรไฟล์ Facebook ร้านจริงของคุณ

## โครงสร้างไฟล์

```
firegame-backend/
├── server.js              ← เซิร์ฟเวอร์หลัก (Express)
├── package.json
├── data/
│   ├── products.json      ← ฐานข้อมูลสินค้า (แก้ไขผ่านหลังบ้านเท่านั้น)
│   └── config.json        ← เก็บรหัสผ่านแอดมิน (สร้างอัตโนมัติตอนรันครั้งแรก)
└── public/
    ├── index.html          ← หน้าร้าน
    └── admin.html          ← หน้าหลังบ้าน
```

## หมายเหตุเรื่องความปลอดภัย

- เซิร์ฟเวอร์นี้ออกแบบมาให้รันบนเครื่องของคุณเอง (localhost) สำหรับจัดการร้านส่วนตัว
- ถ้าต้องการเปิดให้เข้าถึงจากอินเทอร์เน็ตจริง (เช่น deploy ขึ้น hosting หรือเปิดพอร์ตสาธารณะ) แนะนำให้ทำเพิ่มเติม:
  - เปลี่ยนรหัสผ่านแอดมินให้คาดเดายาก
  - ตั้งค่า HTTPS
  - เปลี่ยนค่า `SESSION_SECRET` ใน `server.js` (หรือตั้งผ่าน environment variable) ให้เป็นค่าสุ่มที่ไม่มีใครรู้
- ห้ามแชร์ไฟล์ `data/config.json` ให้ใครเพราะมีรหัสผ่าน (เข้ารหัสไว้แล้ว) ของแอดมินอยู่

## รีเซ็ตรหัสผ่านแอดมิน (กรณีลืมรหัส)

ลบไฟล์ `data/config.json` แล้วรันเซิร์ฟเวอร์ใหม่ ระบบจะสร้างรหัสผ่านเริ่มต้น `admin1234` ให้อีกครั้ง (อย่าลืมเปลี่ยนรหัสใหม่หลังเข้าระบบ)
